Display

Tag Name Description
<inline> Inline Can share lines with other elements. Cant change width/height on CSS.
<inline-block> Inline Block Inline-block display combines features of both inline and block elements. Inline-block elements can appear next to each other and we can specify their dimensions using the width and height properties. Images are the best example of default inline-block elements.
<block> Block Fill the entire width of the page by default, but their width property can also be set. Unless otherwise specified, they are the height necessary to accommodate their content.

Position

Tag Name Description
<static> Static The element is positioned according to the normal flow of the document. The top, right, bottom, left, and z-index properties have no effect on this element. Relative:
<relative> Relative The element is positioned according to the normal flow and then offset by the top, right, bottom, and left properties. The space for the element is reserved in the normal document flow, and other content will not move into that space. It's positioned relative to itself, meaning the offset is applied from where the element would be if it were static.
<absolute> Absolute The element is positioned relative to the nearest positioned ancestor (an ancestor with a position other than static). If there are no positioned ancestors, it's positioned relative to the initial containing block (usually the viewport). The element is removed from the normal flow, so other content may move into its space.
<fixed> Fixed The element is positioned relative to the viewport, meaning it stays in the same place even if the page is scrolled. Like absolute, the element is removed from the normal flow.
<sticky> Sticky This is a hybrid between relative and fixed positioning. The element behaves like a relatively positioned element until it reaches a specified threshold (using top, right, bottom, or left), at which point it becomes fixed. It's useful for creating elements that stick to the top of the viewport as the user scrolls down, such as a sticky header.

Terminal

Command Name Description
pwd Print Working Directory Displays the full pathname of the current working directory.
cd Change Directory Changes the current working directory to the specified directory.
touch Create Empty File Creates a new, empty file with the specified name. If the file already exists, the modification time is updated.
mkdir Make Directory Creates a new directory with the specified name.
clear Clear Terminal Screen Clears the terminal screen, removing all previous output. (Clean only what you see, your path through the directory will keep saved)
